## Short bio
I build AI workflow systems for messy product and ecommerce decisions — with bounded scope, evidence traceability, and explicit trust boundaries.

My work spans D2C ecommerce, product ops, and B2B SaaS. These repos reflect the recurring operational pain I have encountered across multi-brand ecommerce platforms and cross-functional product teams: search incidents without diagnostic language, planning docs that drift into execution misalignment, exception cases that expose policy ambiguity, experiment readouts that outrun their evidence, and launch risk that stays invisible until it is an incident.

The repos are built around a single design bet: AI is most useful when it reduces ambiguity, surfaces evidence, and stops before it lies.

## Profile README opening

Most AI portfolios show wrappers, prompts, or demos.

Mine is built around a different bet:
**AI is most useful when it reduces ambiguity, surfaces evidence, and stops before it lies.**

The repos below are not "AI agents for X."
They are bounded workflow systems for:
- ecommerce search diagnostics
- policy-versioned exception handling
- roadmap contradiction detection
- experiment readout critique
- launch-risk visibility

Each repo is grounded in real operational pain patterns from D2C ecommerce and product operations environments.

Each repo includes:
- explicit trust boundaries
- abstention logic
- structured outputs
- evaluation stubs
- human review by design

The PM design decisions — taxonomy scope, evidence requirements, abstention conditions, escalation logic — are the artifact. The v1 classifiers are intentionally bounded and auditable: constrained heuristics are more trustworthy in review workflows than models generating plausible but unverifiable outputs.
